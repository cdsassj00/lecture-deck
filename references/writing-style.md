# 강의안 문체 규칙 (반드시 지킬 것)

이 규칙들은 실제 임원 강의안 제작에서 여러 번 반려당하며 정리된 것이다. 어기면 결과가 거부된다.

## 0. 원문 보존 — 새 멘트 금지

원본 PPTX·문서·HTML이 있는 작업은 **내용 작성이 아니라 교안형 재배치**가 우선이다.

- 원본의 제목, 항목명, 단계명, 표제어, 문장은 가능한 한 그대로 유지한다.
- 누락된 연결 문장이 꼭 필요할 때만 짧은 사실 설명문을 보탠다.
- 사용자가 요청하지 않은 새 관점, 새 메시지, 새 슬로건, 새 결론을 만들지 않는다.
- "더 세련되게"는 문장을 꾸미라는 뜻이 아니라, 레이아웃·위계·간격·폰트 크기·도식 배치를 정돈하라는 뜻이다.
- 원문이 거칠어도 의미를 바꾸지 않는다. 다듬을 때는 오탈자, 띄어쓰기, 조사, 지나친 중복만 최소 수정한다.

## 1. 표지 제목 — 반드시 사용자에게 확인

표지 제목은 사용자가 보는 첫 문장이므로 **반드시 사용자에게 물어 확정한다.**

- 원본 파일명이나 첫 슬라이드 제목이 있어도 표지에 쓸 최종 제목을 임의로 만들지 않는다.
- 사용자가 "표지 제목은 OOO로 확정"처럼 명시한 경우에만 그대로 쓴다.
- 사용자가 제목을 주지 않았으면 작업 전에 "표지 제목을 무엇으로 할까요?"라고 묻는다.
- 표지 제목에 선언적·시적 표현을 새로 만들지 않는다.

## 2. 제목 — 건조한 기능적 명칭

`[구분] 주제 — 기능적 부제` 형식. 한 소주제 묶음에 걸쳐 같은 제목을 반복해도 된다(교안의 챕터 헤딩처럼).

**금지 (비유·상징·수사·슬로건·블로그체·선언적/시적 문장):**
- ✗ "나의 첫 하네스 — 헌법과 스킬을 얹기"
- ✗ "잘 만든 하네스는 어떻게 생겼나"
- ✗ "두 가지가 하나의 이름을 함께 쓰고 있다"
- ✗ "이 말은 성립하는가" (수사적 질문)
- ✗ "프롬프트를 쌓지 마라, 흐름을 설계하라" (슬로건/대구)
- ✗ "한 흐름을 끝까지 동작하게 만든다"
- ✗ "개발자와 기획자의 경계가 서비스 설계 중심으로 이동한다"
- ✗ "데이터가 일하는 방식이 바뀐다"
- ✗ "AI가 업무의 다음 장을 연다"
- ✗ "새로운 전환점이 시작된다"

**권장 (사실·기능 중심):**
- ✓ "[실습 ①] 지침과 스킬 적용 — 적용 전후 비교"
- ✓ "[사례] 황민호 하네스 — 4계층 문서 구조"
- ✓ "[이론] 좁은 의미와 넓은 의미"
- ✓ "[실습] 기후·에너지·환경 AI서비스 제작 스튜디오 — 문제정의부터 배포 시연까지"
- ✓ "[설계] 기후부 AI서비스 권한·데이터 접근 매트릭스 — 역할별 조회와 수정 범위"
- ✓ "강의 요약" (← "오늘 우리가 함께 본 것" 같은 표현 금지)

## 3. 본문 — 합니다체 설명문

- 개념을 **합니다체 설명 문단**으로 푼다. 예: "우리가 쓰는 OO는 사실 …하는 구조입니다."
- 불릿 나열보다 **번호 개념 정의**(이름 + 2~3줄 설명)를 쓴다.
- 요약 슬라이드는 **사실 진술체**(한다체)로: "OO는 두 가지를 가리킨다. …" — 슬로건 금지.
- 원본에 이미 있는 문장을 우선 배치한다. 새 문장은 원문 항목 사이를 설명해야 할 때만 쓴다.
- 문장을 늘려서 빈 공간을 채우지 않는다. 빈 공간은 레이아웃, 폰트 크기, 표, 도식, 여백 조정으로 해결한다.

## 4. 비개발자 눈높이

대상이 비개발자(임원·일반인)일 때:
- 전문용어는 **풀어서** 쓰고 괄호로 원어 병기. 가능하면 "예 — …" 한 줄로 구체화.
- 추상적 동사보다 **구체적 동작**: "전송/정지 판정/회신" → "묻기 / 고르기 / 확인 후 실행 / 알려주기".
- 개발 용어 **금지**: PR·머지·린터·CI·커밋·레포 → 평이한 말로 치환.
  - 린터/CI/검사 → "어기면 멈추는 자동 검사 장치"
  - 코드가 막는 구조 → "통과 못 하면 다음 단계가 안 열리는 관문"
- 비유에 기대지 말 것(이 사용자는 비유 표현을 특히 싫어함). "헌법" 같은 비유어도 "기준 문서"로.

## 5. 메타 문장 금지

수강생이 보는 본문만 쓴다. "강사는~", "수강생은~", "이 강의는 ~방식으로 운영" 같은 **제작자 시점 문장 금지**. 청중 직접 지시가 꼭 필요할 때만 "여러분".

## 6. 폰트 크기

본문이 너무 작으면 반려된다. 1280×720 캔버스에서 **설명문 16.5px, 개념·표·단계 14~15px**(≈10~11pt 체감)을 기준으로 한다.

- 기본값보다 작게 줄이지 않는다.
- 박스 안 텍스트가 적으면 폰트 크기를 키워도 된다.
- 새 문장을 만들어 공간을 채우지 말고, 기존 텍스트의 폰트 크기·행간·여백·정렬을 조정한다.
- 너무 긴 원문은 의미 단위로 나누어 다음 박스나 다음 슬라이드로 보내고, 글자를 무리하게 줄이지 않는다.

## 7. 출처·수치

- 사례·통계는 출처를 `.srcline`에 단다.
- 수치는 `.stat`으로 강조(예: 49.5→79.3, +60%).
